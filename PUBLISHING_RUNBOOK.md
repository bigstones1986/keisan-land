# けいさんランド 発信運用手順

更新日: 2026-07-16

## 目的

X、note、Substackを同時に埋めることではなく、保護者や先生に役立つ内容を、媒体に合う形で継続して届ける。

## 1日の入口

1. `CONTENT_CALENDAR_2026-07-16_TO_07-31.md` で今日の担当と媒体を確認する。
2. Search Consoleの7日と28日を確認する。数字が取れない日は推測しない。
3. 読者の困りごとを1つだけ決める。
4. 今日が公開日でなければ、原稿改善、画像選定、リンク確認、24時間後レビューのいずれかを進める。

## 媒体別の完成条件

### X

- 最初の2行で家庭や教室の一場面が分かる。
- リンクを外しても、小さな学習ヒントとして役立つ。
- リンク投稿は、内容が一致する教材へ直接つなぐ。
- 原則280文字以内、ハッシュタグなし。
- 教材実物が分かる画像を優先し、使えない日は無理に画像を付けない。

### note

- タイトルが読者の疑問へ直接答えている。
- 冒頭で結論を伝え、1段落を2〜3文程度にする。
- 手順、問題例、声かけ、参考資料、教材URLをつなげる。
- Substackの文章を転載せず、検索から読む人向けに書き直す。
- 初回公開時は、原稿の公開予定日と本文の更新日を一致させる。
- 記事内容と一致する横長の見出し画像と、20文字以上の代替テキストを用意する。
- 学年、教科、単元、利用場面から、具体的なタグを3〜7個選ぶ。
- 公開画面で見出し画像の切り取り、本文画像の位置、タグを確認する。

### Substack

- 運営者が迷った点、直した理由、まだ足りない点のどれかを入れる。
- 開発機能の一覧ではなく、読者にとっての意味を中心にする。
- タイトル末尾に `（開発日記 #番号）` を付ける。
- 文末に、記事内容と一致する教材URLと、けいさんランドURLを入れる。

## 公開前ゲート

1. `npm run qa:publishing` を実行する。
2. `AGENTS.md` の100点評価で90点以上か確認する。
3. 原稿の公開管理情報に、対象読者、編集点、今回の編集焦点、24時間後、7日後の日付があるか確認する。
4. 90点以上の理由を、具体的な場面、教材例、確認済みの事実のいずれかで説明できるか確認する。
5. 実在する機能、問題、答え、URLだけを書いているか確認する。
6. Chromeの投稿画面で、アカウント、タイトル、段落、リンクを最終確認する。
7. 外部サービスへの公開ボタンは、人間の確認後に押す。

公開管理情報:

```text
reader: この記事を一番届けたい具体的な読者
editorial_score: 100点評価の合計
editorial_focus: 今回の原稿で直した1点
review_24h: 公開予定日の翌日
review_7d: 公開予定日の7日後
```

編集点は公開結果ではなく、公開候補としての品質を表す。点数が高くても、公開後の反応を成功と決めつけない。

## 公開記録

`PROJECT_STATUS.md` に次を残す。

```markdown
公開日時:
媒体:
タイトル・投稿冒頭:
リンク先:
公開URL:
公開時に変えた点:
24時間後に見る数字:
7日後に見る数字:
```

## 24時間後

- X: 表示、いいね、返信、プロフィール遷移、確認できる場合はリンククリック
- note: ビュー、スキ、コメント、教材への反応
- Substack: 開封、反応、購読、教材への反応
- 数字が少なくても、1本だけで文体や方針を大きく変えない。

## 7日後

1. 同じ曜日のSearch Consoleと比較する。
2. 表示回数、順位、CTR、クリックのどこが変わったか分ける。
3. 成功、失敗、未判定に分ける。
4. 次の原稿で変える点を1つだけ決める。
5. 読者が返信や反応で使った言葉を、次の原稿のタイトル、冒頭、教材説明の候補へ戻す。

## 禁止事項

- 同じ本文を3媒体へ転載する。
- 投稿数を埋めるために一般論を増やす。
- 成績、苦手克服、検索順位を保証する。
- 子どもの個人情報や学習記録を投稿する。
- 外部投稿や広告費使用を自動実行する。
