# Substack編集長フロー

目的:

- Substack記事を毎回ゼロから考えない
- Search Consoleレポート、今日の作業記録、サイト改善内容から下書きの骨子を作る
- 公開は社長確認を残し、実際にない機能や言い過ぎを防ぐ

## 基本方針

- 完全自動公開はしない
- 自動化するのは「ネタ整理」「タイトル案」「本文下書き」「文末案内」「X投稿案」まで
- Chrome連携でSubstackへ下書きを入れる場合も、公開ボタンは社長が押す

## 毎回見る材料

1. `PROJECT_STATUS.md`
2. 最新の `search-console-private/reports/YYYY-MM-DD.md`
3. その日に公開または改善したページ
4. Search Consoleで見えた変化
5. 社長の言葉、違和感、悩み

## 記事の型

### タイトル

末尾に必ず付ける:

```text
（開発日記 #番号）
```

### 本文の流れ

1. 今日の入口
   - 社長が感じた困りごと、または今日の作業のきっかけ
2. 何を直したか
   - 専門用語を使わずに説明する
3. なぜ大事か
   - 保護者、先生、子どもにとっての意味を書く
4. まだできていないこと
   - 正直に書く
5. 次にやること
   - 小さく、具体的に書く
6. 文末案内
   - けいさんランド名とURLを必ず入れる

## 文末案内

毎回、最後に入れる:

```text
けいさんランドでは、小学生向けの無料計算ドリルを少しずつ作っています。
登録不要で、スマホでも印刷でも使えます。

けいさんランドはこちら:
https://keisan-land.netlify.app/
```

ステップ計算を紹介する回ではこちらも入れる:

```text
毎日ステップ計算はこちら:
https://keisan-land.netlify.app/step-calculation.html
```

## 編集長チェック

- 初心者がスマホで読んでも最後まで止まらないか
- 1文が長すぎないか
- 専門用語が残っていないか
- AIっぽいまとめ口調になっていないか
- 実際に存在しない機能を書いていないか
- 不安を煽っていないか
- 文末URLがクリックできる形になっているか

## 今日の候補

タイトル案:

```text
毎日ちょっとずつ進める計算ドリルを、1年生から3年生まで広げました（開発日記 #11）
```

- 毎日ステップ計算を3年生まで広げた
- あまりのあるわり算を、商とあまりに分けて入力できるようにした

- 手動デプロイから自動デプロイへ進んだ

- 「派手なAI機能」より、親が安心して子どもに使わせられる地味な改善を積み上げる話
